A regular expression is a sequence of characters that specifies a search pattern in text.

As a regular expression "matches" a string it consumes that portion of the string and removes it from consideration for further matchings. For example, "abcde".match(/.../) will match, "abc" but in doing so in consumes that portion of the string which just leaves "de" which is not a match for "...". "abcde".match(/.../) will match, "abc" and "def". In both of these examples, "bcd" and "cde" where never considered even though they are both three characters.

Page #: Assertions

  • "^" matches the beginning of input. The pattern to match must appear after the caret symbol as in "^a".

    If the multiline flag, '/m', is used, then it also applies immediately after a line break. Note that the caret symbol, '^', means negation when used at the start of a range.

  • "$" matches the end of input. The pattern to match must appear before the dollar sign symbol as in "a$".

    If the multiline flag, "/m", is used, then it also applies immediately before a line break.

  • "\b" matches a word boundary. This is the position where a word character is not followed or preceded by another word-character.

    To be precise, \b[pattern] will match a word starting pattern whereas [pattern]\b will match a word ending pattern

  • "\B" matches a non-word boundary. This is a position where the previous and next character are of the same type: Either both must be words, or both must be non-words.

    To be precise, \B[pattern] will require that the beginning of the pattern be at a non-word boundary whereas [pattern]\B will require that the end of the pattern be at a non-word boundary. Compare this example, to this example, .

  • Lookahead assertion x(?=y) matches x only if x is immediately followed by y.
  • Negative lookahead assertion x(?!y) matches x only if x is not immediately followed by y.
  • Lookbehind assertion (?<=y)x matches x only if x is immediately preceded by y.
  • Negative lookbehind assertion (?<!y)x matches x only if x is not immediately preceded by y.

Page #: Character Classes

  • "." matches any single character except line terminators; however, ES2018 added the "\s", "dotAll", flag, which allows the dot to also match line terminators. Compare this example, to this example, . Note that the character class [^] will also achieve selection of any character.
  • \d matches any digit (Arabic numeral).
  • \D matches any character that is not a digit (Arabic numeral).
  • \w matches any alphanumeric character from the basic Latin alphabet, including the underscore. Equivalent to [A-Za-z0-9_].
  • \W matches any character that is not a word character from the basic Latin alphabet. Equivalent to [^A-Za-z0-9_].
  • \s matches a single white space character, including space, tab, form feed, line feed, and other Unicode spaces.
  • \S matches a single character other than white space.
  • \ indicates that the following character should be treated specially, or "escaped". It behaves one of two ways.
    • For characters that are usually treated literally, indicates that the next character is special and not to be interpreted literally.
    • For characters that are usually treated specially, indicates that the next character is not special and should be interpreted literally.
  • There are others. See MDN for more information about them.

Page #: Groups and ranges

  • x|y matches either "x" or "y"
  • [xyz] or [a-c] is a character class as indicated by the brackets. Matches any one of the enclosed characters.

    You can specify a range of characters by using a hyphen, but if the hyphen appears as the first or last character enclosed in the square brackets it is taken as a literal hyphen to be included in the character class as a normal character.

  • [^xyz] or [^a-c] is a negated or complimented character class. That is, it matches anything that is not enclosed in the brackets.
  • (x) is a capturing group. Matches x and remembers the match.

    A regular expression may have multiple capturing groups. These capturing groups can be referenced ordianally or they can be named and referenced by name.

Page #: Quantifiers

  • x* matches the preceding item In this article, item refers not only to singular characters, but also includes character classes, Unicode property escapes, groups and ranges. "x" 0 or more times. One might conclude that x* will match everything because of the "zero" reference, but that is not the case. The "zero" reference only makes sense in the context of a larger item. "Zero" or more relative to a larger context.
  • x+ matches the preceding item "x" 1 or more times. x+ is equivalent to x{1,}
  • x? matches the preceding item "x" 0 or 1 times
  • x{n} where "n" is a positive integer, matches exactly "n" occurrences of the preceding item "x".
  • x{n,} where "n" is a positive integer, matches at least "n" occurrences of the preceding item "x". Note that the "," cannot be followed by any white space.
  • x{n, m} where "n" is 0 or a positive integer, "m" is a positive integer, and m > n, matches at least "n" and at most "m" occurrences of the preceding item "x".
